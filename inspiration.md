# Inspiration

An append-only library of writing you **admire** — the evidence base for the **Preferred voice** (what you want to sound like), the positive twin of [anti-patterns.md](anti-patterns.md). Note hooks, openings, and closings you like here, with the link and what works about them.

**Scope — hooks, starts, and endings/CTAs ONLY. Never whole posts, never the body.** A hook-first reading model: the line(s) before "See more" decide the click, then the body, then the close. So this file collects exactly the parts that decide that: the **hook** (and a few extra lines of the **start** when worth flagging), and the **ending/CTA**. The **body** is not sampled here — it follows your sentence-level preferences in `voice.md`.

**When it's read:** consulted at the **hook and closing step of every draft** — not for every sentence (that's `voice.md` + `guardrails.md`). It calibrates *how an opening or closing feels*; it never picks the post type or hook type (that's `post-types-and-hooks.md`).

The onboarding interview seeds this from any writing you said you admire. Add more whenever you read something that makes you think "I want to open like that."

## How the skill uses this file (exact — the skill must not improvise)

**Role:** this file **CALIBRATES**, it never **DECIDES**. [post-types-and-hooks.md](post-types-and-hooks.md) is the only file that picks the post type and hook type. This file only tunes *how* an already-chosen opening or closing feels.

**Authority order (top wins):** voice.md + guardrails.md (every sentence) → post-types-and-hooks.md (type + hook) → inspiration.md (open/close shape only).

**Procedure — runs only at the opening/closing drafting step:**
1. post-types-and-hooks.md selects the post type + hook type first.
2. The skill checks this file for a saved entry tagged with that matching shape.
3. If found, it reads the entry's **extracted-pattern note** (not the saved text).
4. It shapes the draft to that pattern using **only your raw thought + gap answers** for the words.
5. It tells you which saved example the shape was modeled on (vetoable).
6. If no match, or this file is empty → it does nothing extra and falls back to the generic playbook. Always optional.

**Never:**
- Never copy phrasing, sentences, examples, or ideas from a saved piece into a draft — **pattern only** (enforces the guardrails.md HARD RULE "extract pattern, never reproduce phrasing").
- Never let this file override the type/hook choice.
- Never invent a pattern that isn't written in the entry's note — if an entry has no pattern note, ignore it.
- Never let it affect *substance* — form only; every idea traces to your own raw thought.

## Entry format

Copy this block for each new example. The **extracted pattern** is the only field the skill acts on.

```
### [short label] — [date]
**Type / shape:** [hook | opening | closing] · [which hook type, if known]
**Source / link:** [url]

**The example:**
> [paste the admired hook / opening / closing]

**What I like about it:**
- [the specific thing that works]

**Extracted pattern (skill uses ONLY this):**
- [the reusable shape, in your words — NOT the phrasing. e.g. "state the concrete shallow claim, then the gap, in 3 short lines"]
```
