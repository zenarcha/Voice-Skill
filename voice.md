# Voice Profile

> This file is empty until you run the one-time setup. On first use the skill reads the
> sentinel on the line above; while it says `empty`, the skill runs `onboarding.md` to build
> this profile with you, then flips it to `built`. Do not delete the sentinel line.

The evidence base for how you write (Observed voice) and what you're reaching toward (Preferred
voice). SKILL.md reads this on every draft. Sentence-level rules live in
[guardrails.md](guardrails.md); this file is the voice they protect.

## Preface — how this file is built (read before trusting any entry)

- **Never invent a voice trait.** Every entry below must trace to a real interview answer or an
  elicited writing sample. Nothing is extrapolated.
- **Each entry shows its evidence** — a verbatim line, a sample, or the interview moment it came
  from.
- **Observed vs Preferred are separated.** Observed = how you actually write now (from samples).
  Preferred = what you want to sound like (from the interview and admired writing).
- **Frequency labels govern enforcement:** **HARD RULE** = never violate · **STRONG TENDENCY**
  = ~70-80%, breaking it occasionally is fine · **LIGHT PREFERENCE** = tiebreaker.

## How to use this document (anti-overfitting)

Spirit over letter — internalise the sensibility, don't mechanically stack every tendency.
Respect the frequency labels. Introduce natural variation (don't open every post the same way).
**Litmus test before publishing:** *"Does this sound like something I'd actually write, or like
AI trying hard to imitate me?"* If it feels forced, pull back.
