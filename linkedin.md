# LinkedIn Format Layer

Read this file only when the platform is LinkedIn. It holds LinkedIn-specific facts — format options, platform rules, and best practices. It does **not** make decisions and does **not** touch voice. SKILL.md is the controller; this file is the reference it consults.

Voice (sentence-level style, tone, fragment habits) lives in voice.md and applies on top of everything here. Sentence-level writing rules live in guardrails.md. Nothing in this file overrides those — if a best practice below ever conflicts with the voice profile, the voice profile wins.

---

## How SKILL.md uses this file

Steps 2-3 apply in Draft mode (when a post is actually written). In Outline mode, only Step 1 runs, and it is advisory — the format is a suggestion, not a confirmed choice.

- **Step 1 — Format selection**: consult the Format Options table, match the raw content type to a format. Draft mode confirms it before drafting; Outline mode offers it as advisory direction.
- **Step 2 — Drafting** (Draft mode, during): keep the Best Practices active while shaping the post.
- **Step 3 — Pre-return check** (Draft mode, after drafting, before returning the draft): run the Hard Rules. Fix or flag any failure — never silently return a draft that breaks a hard rule.

---

## 1. Format Options

Match the raw content to the format that fits it. This is a reference, not a forced choice — if the content clearly calls for something else, SKILL.md decides.

| Raw content type | Format |
|---|---|
| Step-by-step process, framework, or checklist | Document post (PDF carousel) — 5-10 slides, one key point per slide |
| Data, benchmarks, or before/after | Document post, or image + text |
| Short insight or opinion | Text post (~150-300 words) |
| Lived moment or behind-the-scenes | Text post |

Notes:
- Document posts (carousels) carry the highest engagement on LinkedIn and are the default for anything educational or structured.
- Carousel specs: 1080x1080 (square) or 1080x1350 (portrait), exported as PDF. Engagement drops after ~10 slides.
- **Carousel output:** the skill drafts a carousel as slide-by-slide text — one labelled block per slide (Slide 1, Slide 2, …), one key point per slide, with the hook on Slide 1. It does not produce the visual design; the user takes the slide text into their design tool.

---

## 2. Hard Rules

Enforced at the pre-return check (Step 3). These are things LinkedIn actively penalizes — non-negotiable.

- **No external links in the post body.** Links cut reach by ~60%. If a link is genuinely required, it goes in the first comment — and reach is still reduced, so flag that to the user.
- **No engagement bait** (e.g. "Agree? Comment 👇"). Classified as spam by the algorithm.

---

## 3. Best Practices

Kept active during drafting (Step 2). Recommendations, not hard rules — apply them unless the content or voice profile says otherwise.

- **Hook in the first 150 characters** (everything before "See more"). The algorithm tests new posts against 2-5% of the network first; a weak opening kills distribution before it starts. The hook surfaces what's already strongest in the raw content — it does not invent.
- **Short paragraphs** — 3-4 lines maximum. White space aids mobile readability.
- **One clear takeaway** per post.
- **Earn a save.** Saves and shares are the signals the 2026 algorithm weights most. The post should contain something worth saving — a framework, a checklist, an original insight — not just a take. If it doesn't, flag that before returning.
- **Hashtags: 3-5 maximum**, preferably in the first comment.
- **Avoid bro-etry** — don't default to stacking one-line paragraphs for effect.
- **Avoid generic motivational language** ("game-changer," "this changed everything," "hustle").
